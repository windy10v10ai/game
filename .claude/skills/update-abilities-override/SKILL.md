# Update Abilities Override

在 Dota 2 版本更新后，把**本图相对当前参考的差分**写进 `game/scripts/npc/npc_abilities_override.txt`（扩展等级、有意加强等）。**引擎会从原版合并缺失键**；override **只应出现与参考相比有改动的键**（见「同值不保留」）。

## 维护思路（总览）

1. **定本图规则**：从既有 override 行尾注释与数值读出设计意图（如 `x2`、`+100`、延伸步长）；换参考版本后，在**新官方数**上按**同一规则**重算本图 `value`，**不得**只改注释、让数值与规则脱节。
2. **先删同值**：与参考逐字相同的整键（含多档但每档与参考对应档相同、或「凑末档重复」）整段不保留。
3. **再补差分与档数**：对**确有本图差分**或须延伸的多档键：若本图各等级意图**同一数值**，只写**单 token**（见下「恒定单值」），禁止为对齐 `MaxLevel` 写 `n n n` 重复凑档。对其余**随等级变化**的多档，经「同值多档」判定后仍保留者，档数须与有效 `MaxLevel` 一致。参考里**已是多档**的字段，若本图提高了该技能 `MaxLevel`，通常须在 override **写出延伸档**（见**「MaxLevel 扩展必扫」**）；**不得**因 override 里从未写过该键而跳过。
4. **最后自检**：无同值抄写、无 facet；无「全相同 token」的多档行；**凡本图写了更高 `MaxLevel` 的技能**，按「MaxLevel 扩展必扫」对照**参考全文**逐项打勾，含**参考有而 override 无**的多档键。

## 何时使用

- 大版本（如 7.40、7.41）后维护 `npc_abilities_override.txt`。
- 按英雄逐个对齐官方参考、扩展 `MaxLevel`、延续固定加强或倍数加强。

## 输入与前置条件

用户应尽量提供：

- 参考版本号 `{version}`（与 `docs/reference/{version}/` 目录一致）。
- 待处理英雄列表（按顺序逐个）。
- GitHub Issue URL（可选，用于进度追踪）。

确认存在：

- `docs/reference/{version}/npc_heroes.txt`（英雄 `Ability1`–`Ability6` 槽位，见下节「未指定技能时」）。
- `docs/reference/{version}/heroes/npc_dota_hero_{hero}.txt`（各技能 KV 官方块，本仓库内多为 `DOTAAbilities` 切片）。
- `docs/reference/{version}/abilities_schinese.txt`（可选，用于表格/说明可读性）。

```bash
grep -n "ability_name" game/scripts/npc/npc_abilities_override.txt
grep -n '"npc_dota_hero_antimage"' docs/reference/{version}/npc_heroes.txt
```

## 未指定具体技能时：用 `npc_heroes.txt` 定范围

用户**未列出**要改的技能名时，在 **`docs/reference/{version}/npc_heroes.txt`** 中定位该英雄块，读取槽位，并在 **`npc_abilities_override.txt`** 中**逐个核对**同名技能：仅保留**确有本图差分**的技能块；无差分的技能**不写整块**。

### 必看的四个槽位

以下槽位在常规英雄上对应「三小技能 + 大招」，**一律**纳入本次更新范围（除非用户明确排除）：

| 槽位 | 含义（典型） |
| ---- | ------------ |
| `Ability1` | 一技能 |
| `Ability2` | 二技能 |
| `Ability3` | 三技能 |
| `Ability6` | 大招（非 `Ability4`） |

示例（敌法师）：

```kv
"Ability1"		"antimage_mana_break"
"Ability2"		"antimage_blink"
"Ability3"		"antimage_counterspell"
"Ability6"		"antimage_mana_void"
```

### `Ability4` / `Ability5`：仅在为实技能时处理

- 若为 **`generic_hidden`**：视为**空槽**，**不必**为「占位」单独写 override。
- 若**不是** `generic_hidden`：值为真实技能名时，该技能**同样**须按参考同步进 `npc_abilities_override.txt`（与 `Ability1`–`3`、`Ability6` 相同规则）。

> **自定义英雄**：若 `npc_heroes_custom.txt` 覆盖了某槽位技能名，以 **custom 槽位**为准再解析要维护的 override 技能列表（与 `update-heroes-custom` 槽位合并规则一致）。

## 核心原则

**优先级（维护任一技能时按此顺序想，避免和下面各条打架）：**

1. **先**做「同值」判定（含**同值多档**，见下表）：与参考**逐字相同**的单值、或多档中**每一档**与参考**对应档**数值相同（含参考只有 3 档、本图 `MaxLevel` 为 4 却把第 4 档写成与前几档相同的「凑档」，如 `20.0`×4 且参考为 `20.0`×3）——**一律视同与原版无差分，整键删除**，**不得**为对齐本图 `MaxLevel` 而在 override **补写或拉长**该键。
2. **再**处理确有本图数值差分的键：固定加、倍数、延伸**且**新档相对官方有**实质变化**等；并对照**「多档子键与 MaxLevel」**与**「MaxLevel 扩展必扫」**检查参考里是否还有**未写入 override**、但随等级变化且本图已提高 `MaxLevel` 的多档字段（含子块 `value`、顶层平铺、块顶多档）。
3. **最后**对「仍保留在 override 里的键」核对：**多档且各 token 并非全部相同**时，档数须与本技能有效 `MaxLevel` 一致（见下「多级数值与 MaxLevel」）；若本图各等级**恒为同一数**，则**只写单 token**（见「恒定单值」）。**已按第 1 步删除的键，不要求在 override 里补档。**

### 多档子键与 MaxLevel（易漏项）

- **恒定单值（禁止重复多档）**：本图对该键的意图是**所有等级同一数值**时，override **只写一个 token**，例如 `"value" "60"`，**禁止**写成 `"60 60 60 60"` 仅为对齐 `MaxLevel` 或「看起来多档」。写入多档前自检：按空格拆分后若**全部 token 相同** → 改为单值。与「同值多档」区别：同值多档指与**参考**逐档相同或末档重复凑档；恒定单值指本图**刻意**全等级同数。
- **MaxLevel 扩展必扫（强制）**：本图 `MaxLevel` **高于**参考默认等级数时，在写完/保留 override 前**必须**通读参考中该技能**整段**，以参考为**全集**列出**所有**「空格分隔的多档数值」键位（块顶 + `AbilityValues`），**禁止**只检查 override 里已出现的键。
  - 将上述清单与当前 override 同技能块**做差集**，标出 **「参考有而 override 无」** 的多档键。换参考后官方**新增**的子键或块顶多档，若本图抬了 `MaxLevel` 且该键**随等级变化**，须按优先级表判定是否在 override **新写**该键的差分/延伸；**禁止**「override 里从来没写过就不管」。
  - 对每个多档键（含差集内新出现的）：先 **同值多档** → 再 **恒定单值**（全同则单 token）→ 其余再谈档数与 `MaxLevel` 对齐。
  - **块顶多档**：`AbilityCooldown`、`AbilityDamage`、`AbilityManaCost`、`AbilityDuration`、`AbilityCastRange`、`AbilityCastPoint` 等任一为 `"n n n"` 形式。
  - **`AbilityValues` 内子块**：`"某键" { "value" "a b c" ... }`。
  - **`AbilityValues` 内顶层平铺**（最易漏）：与子块同级、**直接** `"键名" "a b c"` 的行（无子块），例如 `damage_per_second`、`speed` 等；**不得**只扫带 `value` 子块的键。
  - **典型漏项**：`ancient_apparition_ice_blast` 的 `damage_per_second`（参考 `12 24 36`，本图大招 `MaxLevel` 4 须写第四档）。
  - **正例（参考新增平铺）**：参考在 `AbilityValues` 内新增顶层平铺 `"foo" "10 20 30 40"`，本图小招 `MaxLevel` 5、且对该技能已有差分或延伸会得到**新数**：可仅在 override 写 `"foo"` 五档（按相邻差延续第五档），并 `//` 对照参考前四档；若延伸第五档仅重复末档且无设计意图，则整键不写，见「同值多档」。
- 通读参考中该技能的 `AbilityValues`：若子键 `value` 为**空格分隔多档**，档数通常等于该技能参考**默认等级数**。本图对该技能写了更高 `MaxLevel` 时：
  - 延伸新档只会**重复参考末档**、且无单独设计意图 → 仍属「同值多档」，**不写** override。
  - 官方各档**互不相同**且本图已对该技能有差分或明确要随等级延伸 → 通常须在 override **只写该子键**的 `value`，**写满与有效 `MaxLevel` 相同档数**；新档按官方**相邻档差**延续（例：`800 1200 1600 2000` 每档 +400 → 第五档 `2400`），除非落入「数值情形速查」末档封顶等特例。
- **单档与多档并存**：同一技能可有单档子键（如磁场 AOE `radius` 单值 `300`）与多档子键（如 `rune_pull_radius` 四档）并存。**勿**仅因 `MaxLevel` 为 5 就把单档键硬拉成五档，除非确有差分并在块首 `//` 说明。

### 行尾注释 = 换算规则（与数值必须一致）

- 行尾 **`x2`、`(x1.5)`、固定 `+N` 差值**等表示可重复的换算规则。参考版本更新后：先将 `//` 后对照段更新为**当前参考**官方数，再按**同一规则**重算引号内本图数值。
- **禁止**：发现数值与 `x2` 等不符时**只删倍数标记**、保留错误数字；应**要么**改数值以符合规则，**要么**放弃该规则并改为能描述真实曲线的写法（见下「行尾注释」表）。

| 原则 | 说明 |
| ---- | ---- |
| **差分-only** | override **不是**「把参考里的技能再抄一遍」。凡在参考中**已存在且与参考逐字相同**的键（含子键），**一律不得**出现在 override 中；由原版 KV 提供即可。 |
| **同值不保留（须严格执行）** | 含但不限于：`AbilityBehavior`、`AbilityUnitTargetTeam` / `Type`、`SpellImmunityType`、`SpellDispellableType`、`AbilityType`、`FightRecapLevel`、`AbilitySound`、`IsShardUpgrade`、`IsGrantedByShard`、`HasScepterUpgrade`、`Innate`、`AbilityCastAnimation`、**与参考相同的** `AbilityCastPoint` / `AbilityCastRange`、`AbilityChannelTime`、`DamageTypeTooltip`、`affected_by_aoe_increase` 等——只要**值与参考完全一致**就**删除**，不要为了「看起来完整」而抄写。 |
| **天赋 / 神杖等同路子键** | `AbilityValues` 内 `special_bonus_unique_*`、`special_bonus_scepter`、`special_bonus_shard` 等：**与当前参考逐字相同**的不得在 override 再写一行；本图只改同一父键下的 `value`（或其它确有差分的子键）时，**勿顺带抄写**未改动的天赋/神杖行。若本图对该天赋**确有不同数值**，只写该差分子键即可。 |
| **`CalculateSpellDamageTooltip`** | override **禁止**出现此键，**不**因「怕合并丢 tooltip」而抄写；交给原版 KV。存量旧稿在维护到该技能时**删除**此行。 |
| **本图仍须写明的情形** | 仅当：**至少一档数值**与参考不同、`MaxLevel` 扩展且伴随**实质数值变化**、或**子树中某一枝**与参考不同时，才写对应键；**仅**比参考多一档但**各档数字与参考对应档仍相同**的，不算「须写明」，见「同值多档」。可只写变化的子键，**不要**顺带贴上同值的兄弟键。 |
| **官方已删** | 参考中已删除的键，override 内不得再保留。 |
| **`special_bonus_facet_*` 已废（7.41+）** | override **禁止**再写。参考里若某 `AbilityValues` 子键**仅有**已废 facet、**没有**顶层 `value`**，视为官方当前无此数值通道——**不得**在 override **新造** `value` 续命；**删除** override 中该键整条。若官方把同一效果挪到别的键或天赋，再按新结构写差分。 |
| **同值多档** | 多档 `value`（或同级结构）若**每一档与参考对应档的数值相同**，**视同单值与原版一致**：整键**不写入** override、已写入的**删除**，交给原版合并。**优先于**「本图 `MaxLevel` 更高所以要补全档数」——**禁止**为凑档而把 `125`×4 抄成 `125`×5，或把 `AbilityDuration` 参考 `20.0`×3 抄成 `20.0`×4（各档仍 20）。**仅当至少有一档与参考对应档不同**（本图加强或规则化延伸出**新数**）时，才在 override 写该键，并写满与有效 `MaxLevel` 一致的档数。 |
| **恒定单值** | 本图**刻意**令各等级为**同一数**时，只写**单 token**，禁止写成多个相同 token；与「多档条数对齐 `MaxLevel`」**不矛盾**——后者仅适用于**真正随等级变化**的多档曲线。 |
| **键集合** | 原则上**不增加**参考里同名技能**没有的键**。若本图确需增键，在**该技能块上方**用 `//` **中文说明原因**即可。 |
| **多级数值与 MaxLevel** | **仅适用于**经「同值多档」判定后**仍须保留**、且**各档 token 并非全部相同**的多档字段：其档数须与本技能有效 `MaxLevel` 一致。恒定单值键用单 token，不要求凑满档数。若整键已因同值删除，**不要求**在 override 为该键补档。未出现在 override 的字段仍由原版提供。无 `MaxLevel` 时按下文「缺省 MaxLevel」推断。 |

### `AbilityValues` 内多子键时的差分-only

- 参考里已有、与本图**逐字相同**的平铺键或子块：**不写**进 override，**勿**为「和参考块对齐」或「把原版四档抄全」而照抄。
- **仅**对存在**实质差分**的子键写 override。典型：本图 `MaxLevel` 高于参考该键的多档档数，且延出**新数**、非重复参考末档凑数时，只写**这一条**子键的 `value`，写满与有效 `MaxLevel` 一致的档数；前几档可与参考对应档相同，因整行相对参考已属延伸后的差分。
- 该子键块内：与参考相同的 `special_bonus_*` 等**仍不写**；**其它**子键无差分则同样不写。

## 行尾注释

与块首 `// 中文技能名` 等搭配使用；**仅在本图数值与参考不一致**（或需说明倍率/差值/设计意图）时，在**保留的键行**上写行尾注释。

| 用途 | 示例 |
| ---- | ---- |
| 单值相对原版 | `"AbilityCooldown" "12" // 18` |
| 多档相对原版 | `"90 80 70 60" // 150 130 110` |
| 倍数 | `"value" "30 60 90 120 150" // 15 30 45 60 x2`（本图五档时先按官方步长推出第五档官方值再 × 倍率）；`x2`、`(x1.5)` 写在 `//` 段**末尾**，紧接参考官方数之后 |
| 固定加层 / 差值说明 | `"value" "300 500 700 900" // 200 400 600 800 +100`、`"+30" // 10 +20差值` |
| 纯说明（无数字） | `// 白送天赋`、`// 与镖同步`、`// 百分比伤害，限制上限` |

**惯例**：

- **与参考同值**：不要在 override 里留「仅备忘」行；**删除该键这一行**（不要用 `"30" // 30` 等形式占坑）。若删完后该技能块已与参考无任何本图差异，按项目习惯可整段去掉该技能 override。
- **有改动、需对照或说明倍率/差值**的键行保留，并写行尾注释，便于下次版本同步时重算。
- **`//` 后对照数**：先写 **当前参考里该键的官方数值**（单值或多档空格分隔），再写相对该对照的倍率或差分标记（如 `+100`）。**倍数**须作为 `//` 段**最后一项**接在参考数之后（如 `// 15 30 45 60 x2`）；不得只有倍数、不写参考数。参考里若**没有**顶层 `value`、仅在已废弃的 `special_bonus_facet_*` 下有过数，**只写那个数字作对照**，**不要**在注释里写「命石」「facet」或旧子键名。**不要**用分号再接「五档」等档位备忘，`MaxLevel` 与多档 `value` 已表达档数。
- **参考换新版本后**：仍保留的键上，行尾**对照数**须更新为**当前参考**里该键的官方值；本图 `value` 按上文「行尾注释 = 换算规则」在**新参考官方数**上重算，**勿**让注释规则与引号内数值不一致。

## 本图等级规则

**先按参考里的 `AbilityType` 区分：**

- **大招**：`AbilityType` 为 `ABILITY_TYPE_ULTIMATE`。
- **小招**：**非** `ABILITY_TYPE_ULTIMATE` 的其它技能（常规技能）。

**本图目标 `MaxLevel`：**

| 类型 | `MaxLevel` |
| ---- | ---------- |
| 小招 | 5 |
| 大招 | 4 |
| 特殊技能 | 单独与策划/注释约定 |

### 缺省 MaxLevel（仅当参考块与 override 均未写 `MaxLevel` 时，用于核对参考里多级 value 档位数）

| 条件 | 推断有效等级 |
| ---- | ------------- |
| 大招（`ABILITY_TYPE_ULTIMATE`） | 3 |
| 小招（其它 `AbilityType`） | 4 |

若任一侧已写明 `MaxLevel`，以**显式值**为准；本图扩展仍以「小招 5 / 大招 4」为默认目标，除非另有约定。

## 数值情形速查（本图相对官方）

处理某英雄前，先判定每个相关技能属于哪一类，**整技能保持一致策略**。

### 1. 仅等级扩展（无加强）

- **先做「同值多档」判定**：若延伸后各档仍与参考已有档**同数值**（例如官方三档全 `20`、本图第四档仍 `20`），属于**无实质差分**，**删整键**，不按本节补档（见核心原则优先级第 1 步）。
- 仅当新档相对参考末档按**等差或其它规律得到新数**（与参考任一档不同）时：前若干档与官方相同；新档按官方等差（或明显规律）延伸。
- 注释可写**新档对应的参考档**（空格分档数字），风格与邻行一致即可，例如扩展后整行后附 `// 80 110 140 170` 表示参考前四档。

### 2. 固定数值加强 / 差值

- 官方更新后保持**固定差**或备注中的 **+N 差值**（参见 `// 10 +20差值` 类写法）。
- 行尾用 `// +100`、`// 10 +20差值` 等与现有文件一致即可。

### 3. 翻倍或其它倍数

- 各档官方值 × 固定倍率；行尾 **`// x2`** 或 **`// (x1.5)`**。本图 `MaxLevel` 高于参考档数时，**先**按官方相邻差推出「若存在下一档」的官方值，**再**乘倍率，使本图档数与 `MaxLevel` 一致。

### 4. 其它

- 等差重算、单值改多档等：在块首或行尾用**中文**说明意图，沿用文件中 `// …` 说明型注释风格。

### 5. 最大档保持不变的特例

- 过短 CD（小招末级 ≤10s、大招末级 ≤60s）、高百分比、`radius` 等：末档可与参考末档持平，不按等差再抬——见邻区技能写法。延伸多档时若等差会把**末级**压到低于参考末档且落入上述过短区间，最后一档用**参考末档**，勿再下压。
- 参考档数少于本图 `MaxLevel`、冷却需多写档位时：**尽量**整段保持**同一相邻档差**（不必与参考的相邻差相同）。常用为自一合理首档起**每档 −1s** 递减至**参考末档**，末级与参考末级一致，例如参考 `12 10 8 6`、五档可写 `10 9 8 7 6`。避免前几档沿用参考步长、后几档突然重复末档造成等差被打断两次。若仅此等差仍会跌破参考末档或过短区间，再按上条用参考末档打断续写。

## 官方重构时的清理

- 官方删除的键与 KV 结构变化：override 中**删除**对应废弃内容；全文件 `grep special_bonus_facet_` 清理（规则见上表）。
- 遇 `grep CalculateSpellDamageTooltip`：在 `npc_abilities_override.txt` 中**删除**匹配行（本文件不应出现该键）。
- **官方新增键**：对照参考列出相对旧版**新增**的多档或块顶键，纳入**「MaxLevel 扩展必扫」**与**参考/override 差集**，避免只删废弃、漏处理新字段。
- 同步后检查：无同值抄写、无 `special_bonus_facet_*`；无「全相同 token」的多档凑行；**仍保留在 override 内**的**随等级变化**多档键，档数与 `MaxLevel` 一致，或该键为恒定单值；对「本图提高了 `MaxLevel`」的技能，已按**参考全文**做**MaxLevel 扩展必扫**（含**参考有而 override 无**的键、顶层平铺）；行尾对照为**当前参考官方数**、注释无命石/facet 名。

## 推荐执行顺序（逐英雄）

1. **定技能列表**：若用户未给具体技能名 → 从 `npc_heroes.txt`（及必要时 `npc_heroes_custom.txt` 槽位覆盖）读取 `Ability1`–`3`、`Ability6`，并按上节规则决定是否包含 `Ability4`–`5`。
2. **定位**：在 override 与 `heroes/npc_dota_hero_*.txt` 中定位上述技能名。
3. **逐技能机械核对（每个技能必做，禁止跳步）**  
   1. 打开参考中该技能**整段**；`grep` 或片段读取本图 `npc_abilities_override.txt` 中该技能块。  
   2. **列出参考中所有多档键**（块顶多档 + `AbilityValues` 内子块 `value` 多档 + 顶层平铺多档）。  
   3. **列出 override 中已写键**；标出 **「参考有而 override 无」** 的多档键，本条**不得跳过**。  
   4. 对每个待保留或待新增键：先 **同值多档**（删或勿写）→ 再 **恒定单值**（全同 token 则改为单值）→ 其余再核对多档条数与有效 `MaxLevel`。  
   5. 行尾 `//` 对照更新为**当前参考**官方数。
4. **逐键对照参考**：只把**与参考不同**的键写入（或保留）override；**删除**与参考逐字相同的键（含 `AbilityBehavior` 等）。**同一轮内先删「同值 / 同值多档」整键**，再处理延伸与差分。若本图 `MaxLevel` 高于参考默认档数：**必须**执行「**MaxLevel 扩展必扫**」：以参考为全集，**含差集内新键**；**禁止**只处理 override 里已改动的子键。
5. **分类**：无加强 / 固定差 / 倍数等。
6. **改值 + 注释**：沿用**同文件、同英雄区段**已有注释风格。
7. **校验**：见下「维护后自检清单」。
8. **汇报**：仅当用户明确要求时再 `git commit`。

## 不应做的事

- **把参考里与原版一致的键抄进 override**（含 `AbilityBehavior`、免疫/目标类型、`AbilityCastAnimation`、与参考相同的 `special_bonus_*` 天赋/神杖子键等），无论是否为了「结构好看」「怕子块合并丢键」或**在同一 `AbilityValues` 里为补全展示而照抄参考同值子键**——与上表「差分-only / 同值不保留」相反的做法一律避免（**以核心原则表及上节「AbilityValues 内多子键时的差分-only」为准**）。
- 在 override 中写 **`CalculateSpellDamageTooltip`**。
- **行尾注释写命石、facet 名称、已删子键名，或用分号接「五档」等档位备忘**（对照只写官方数及倍率/差分标记，见「行尾注释」惯例）。
- 引入与全文件、同英雄区段**割裂**的注释风格（应沿用邻行既有写法）。
- **无说明**地增加参考中不存在的键。
- 参考已更新后，仍不按行尾规则（含 `x2` 等）重算 `value`，或行尾对照数未与**当前参考**对齐。
- 一次读入整个 `npc_abilities_override.txt`；用 `grep`/片段读取。
- **禁止**为对齐 `MaxLevel` 把**本图各等级恒同**的数值写成 `n n n` 多 token；用**恒定单值**。
- **禁止**做「MaxLevel 扩展必扫」时**只扫 override 已有行**；必须以**参考该技能整段**为全集并做**参考/override 差集**。

## 维护后自检清单

每英雄或每批技能改完后逐项勾选（可默念，不必写入仓库）：

- [ ] 已对照**参考全文**列出该技能**全部**多档键，含 **override 尚未出现**的键。
- [ ] 本图提高了 `MaxLevel` 的，已按上表完成 **MaxLevel 扩展必扫**（含顶层平铺、块顶多档）。
- [ ] 无与参考逐字相同的键；无「全相同 token」的多档凑行；恒定意图已用**单 token**。
- [ ] 无 `special_bonus_facet_*`、无 `CalculateSpellDamageTooltip`；行尾对照为当前参考；增键有块首 `//` 说明。
